[float]
=== 주의와 경고
줄 끝에 세미콜론을 잊으면 컴파일러 에러가 날 거다. 에러 텍스트는 분명하고, 빠진 세미콜론을 참조하거나, 그러지 않을 수도 있다.
뚫을 수 없거나 비논리적 컴파일러 에러로 보이는 컴파일러 에러가 오면, 가장 먼저 확인 할 것 중 하나가 컴파일러가 불평한 바로 전에 빠진 세미콜론이다.
[%hardbreaks]
----
int a = 13;
----
